# Documentation Language Policy

Canonical repository documentation language is English.

This policy is enforced for all new or modified persistent documentation going forward.
Legacy non-English text in existing files must be translated incrementally through follow-up documentation PRs.

User-facing operator guides, methodology documents, schemas, release notes, README files, citation files, and publication materials must use English as the canonical repository language.

Non-English working notes may exist only as temporary local drafts and must not be merged into `main`.

If source material is drafted in another language, it must be translated before merge when that content is added or updated.
